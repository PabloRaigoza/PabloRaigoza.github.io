Creating a Window Using SDL2

Tutorial Lists

So if you have followed the tutorial correctly, you would have installed SDL2. This means that we can now #include SDL into our programs. That is what I have done on the first line of code.

Next you will see that I end up using the standard namespace. But in our int main() you will see that I included some extra parameters. I am not entirely sure what they do or even if I need them, but I have always included them in the past.

The next couple lines code set the dimensions our screen. Then we go ahead and load in some of our SDL into our program. We call the function SDL_Init(). This function loads whatever substystem you would like to load. Here are all the parameters:

Syntax Operation
SDL_INIT_TIMER Loads timer subsystem
SDL_INIT_AUDIO Loads audio subsystem
SDL_INIT_VIDEO Loads video subsystem and events subsystem
SDL_INIT_JOYSTICK Loads joystick subsystem and events subsystem
SDL_INIT_HAPTIC Loads haptic subsystem
SDL_INIT_GAMERCONTROLLER Loads gamecontroller subsystem and joystick subsystem
SDL_INIT_EVENTS Loads events subsystem
SDL_INIT_EVERYTHING Loads all above subsystem

After we loaded our SDL we must make a window to display. We do this using the SDL_Window object. This creates us a window to use. Next we assign our object equal to the function SDL_CreatWindow(). This function takes in six parameters.

  • The first parameter is the a char array or string of whatever you would like your window to be called.
  • The next two parameters are the x and y cordinates of where you would like your window to appear. I just used SDL_WINDOWPOS_UNDEFINED for both my x and y cordinates.
  • The next two parameters are your width and height of the window. I used the constant integers we assigned before.
  • The last parameter is the state of the window. I simply used SDL_WINDOW_SHOWN. For these tutorials this should suffice.

The next line is important because here is where we put delay into the program. The delay is important because otherwise the program would run so fast that you could not see the window pop up. Please note that the function SDL_Delay() delays the program by the number of mili-seconds you provide. Lastly we must simply free the resources that we have used. This required to call the function SDL_DestoryWindow() and SDL_Quit(). It is also good practice to assign your SDL_Windows to NULL when you are done using them.
